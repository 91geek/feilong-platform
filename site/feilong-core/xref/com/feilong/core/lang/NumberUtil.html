32  /**
33   * 处理{@link Integer},{@link Long},{@link BigDecimal}等数据类型.
34   * 
35   * <h3>{@link RoundingMode#HALF_UP}与 {@link Math#round(double)}的区别:</h3>
36   * 
37   * <blockquote>
38   * <p style="color:red">
39   * 注意{@link RoundingMode#HALF_UP} -2.5 会变成-3,如果是 {@link Math#round(double) Math.round(-2.5)} 会是-2
40   * </p>
41   * </blockquote>
42   * 
43   * <h3>{@link Double}转{@link BigDecimal}:</h3>
44   * 
45   * <blockquote>
46   * <p>
47   * 对于 double 转成 BigDecimal,推荐使用 BigDecimal.valueOf(double),不建议使用new BigDecimal(double),参见 JDK API
48   * </p>
49   * <ol>
50   * <li>new BigDecimal(0.1) {@code ==>} 0.1000000000000000055511151231257827021181583404541015625</li>
51   * <li>BigDecimal.valueOf(0.1) {@code ==>} 0.1</li>
52   * </ol>
53   * <p>
54   * 在《Effective Java 》这本书中也提到这个原则,float和double只能用来做科学计算或者是工程计算,在商业计算中我们要用 {@link java.math.BigDecimal}.
55   * </p>
56   * </blockquote>
57   * 
58   * <h3><a name="RoundingMode">JAVA 8种舍入法:</a></h3>
59   * 
60   * <blockquote>
61   * 
62   * <table border="1" cellspacing="0" cellpadding="4" summary="">
63   * 
64   * <tr style="background-color:#ccccff">
65   * <th align="left">字段</th>
66   * <th align="left">说明</th>
67   * </tr>
68   * 
69   * <tr valign="top">
70   * <td>{@link RoundingMode#UP}</td>
71   * <td>远离零的方向舍入. 向绝对值最大的方向舍入,只要舍弃位非0即进位.</td>
72   * </tr>
73   * 
74   * <tr valign="top" style="background-color:#eeeeff">
75   * <td>{@link RoundingMode#DOWN}</td>
76   * <td>靠近零的方向舍入,向绝对值最小的方向输入,所有的位都要舍弃,不存在进位情况.</td>
77   * </tr>
78   * 
79   * <tr valign="top">
80   * <td>{@link RoundingMode#CEILING}</td>
81   * <td>靠近正无穷方向舍入  向正无穷方向舍入 向正最大方向靠拢.<br>
82   * 若是正数,舍入行为类似于ROUND_UP,<br>
83   * 若为负数,舍入行为类似于ROUND_DOWN.<br>
84   * <span style="color:red">Math.round()方法就是使用的此模式.</span></td>
85   * </tr>
86   * 
87   * <tr valign="top" style="background-color:#eeeeff">
88   * <td>{@link RoundingMode#FLOOR}</td>
89   * <td>靠近负无穷方向舍入  向负无穷方向舍入 向负无穷方向靠拢.<br>
90   * 若是正数,舍入行为类似于ROUND_DOWN;<br>
91   * 若为负数,舍入行为类似于ROUND_UP.</td>
92   * </tr>
93   * 
94   * <tr valign="top">
95   * <td>{@link RoundingMode#HALF_UP}</td>
96   * <td>四舍五入,生活中的舍入方法.<br>
97   * 最近数字舍入(5进).<span style="color:red">这是我们最经典的四舍五入</span>.</td>
98   * </tr>
99   * 
100  * <tr valign="top" style="background-color:#eeeeff">
101  * <td>{@link RoundingMode#HALF_DOWN}</td>
102  * <td>五舍六入,最近数字舍入(5舍). 在这里5是要舍弃的.</td>
103  * </tr>
104  * 
105  * <tr valign="top">
106  * <td>{@link RoundingMode#HALF_EVEN}</td>
107  * <td>精确舍入,银行家舍入法. <br>
108  * 四舍六入;五分两种情况,如果前一位为奇数,则入位,否则舍去. <br>
109  * 以下例子为保留小数点1位,那么这种舍入方式下的结果:  <br>
110  * {@code 1.15 返回 1.2} {@code 1.25 返回  1.2}</td>
111  * </tr>
112  * 
113  * <tr valign="top" style="background-color:#eeeeff">
114  * <td>{@link RoundingMode#UNNECESSARY}</td>
115  * <td>无需舍入</td>
116  * </tr>
117  * 
118  * </table>
119  * 
120  * <a name="RoundingMode_HALF_EVEN"></a>
121  * <h3>关于 {@link RoundingMode#HALF_EVEN}:</h3>
122  * 
123  * <blockquote>
124  * 
125  * <pre class="code">
126  * 该算法是由美国银行家提出了,主要用于修正采用上面四舍五入规则而产生的误差。如下：
127  * 
128  * 舍去位的数值小于5时,直接舍去。
129  * 舍去位的数值大于5时,进位后舍去。
130  * 当舍去位的数值等于5时,
131  *          若5后面还有其他非0数值,则进位后舍去,
132  *          若5后面是0时,则根据5前一位数的奇偶性来判断,奇数进位,偶数舍去。
133  * </pre>
134  * 
135  * <b>对于上面的规则我们举例说明:</b>
136  * 
137  * <pre class="code">
138  * 11.556 = 11.56 ------六入
139  * 11.554 = 11.55 -----四舍
140  * 
141  * 11.5551 = 11.56 -----五后有数进位
142  * 
143  * 11.545 = 11.54 -----五后无数,若前位为偶数应舍去
144  * 11.555 = 11.56 -----五后无数,若前位为奇数应进位
145  * </pre>
146  * 
147  * </blockquote>
148  * 
149  * </blockquote>
150  *
151  * @author <a href="http://feitianbenyue.iteye.com/">feilong</a>
152  * @see Integer
153  * @see Long
154  * @see BigDecimal
155  * @see Number
156  * @see NumberPattern
157  * @see RoundingMode
158  * @see org.apache.commons.lang3.math.NumberUtils
159  * @since 1.4.0
160  */
161 public final class NumberUtil{

167         throw new AssertionError("No " + getClass().getName() + " instances for you!");
168     }

174     /**
175      * 获得除法结果<code>one/two</code>,四舍五入{@link RoundingMode#HALF_UP},小数位数 <code>scale</code> 指定.
176      * 
177      * <h3>示例:</h3>
178      * 
179      * <blockquote>
180      * 
181      * <pre class="code">
182      * NumberUtil.getDivideValue(0, 2, 0)   =   toBigDecimal(0)
183      * NumberUtil.getDivideValue(6, 4, 0)   =   toBigDecimal(2)
184      * NumberUtil.getDivideValue(10, 3, 2)  =   toBigDecimal(3.33)
185      * NumberUtil.getDivideValue(5, 3, 2)   =   toBigDecimal(1.67)
186      * </pre>
187      * 
188      * </blockquote>
189      * 
190      * @param one
191      *            除数
192      * @param two
193      *            被除数,自动转成{@link BigDecimal}做除法运算
194      * @param scale
195      *            标度,小数的位数,四舍五入,用于 {@link java.math.BigDecimal#setScale(int, RoundingMode)}<br>
196      *            如果为零或正数,则标度是小数点后的位数。<br>
197      *            如果为负数,则将该数的非标度值乘以 10 的负 scale 次幂 (通常情况用不到负数的情况)
198      * @return 如果 <code>one</code> 是 null,抛出 {@link NullPointerException}<br>
199      *         如果 <code>two</code> 是 null,抛出 {@link NullPointerException}<br>
200      *         如果 <code>two</code> 是 0,抛出 {@link IllegalArgumentException}<br>
201      *         否则转换成{@link BigDecimal} 返回除法结果one/two,四舍五入 {@link RoundingMode#HALF_UP},小数位数 <code>scale</code> 指定
202      * @see <a href="#RoundingMode">JAVA 8种舍入法</a>
203      * @see java.math.RoundingMode#HALF_UP
204      * @see java.math.BigDecimal#ROUND_HALF_UP
205      * @see #getDivideValue(Number, Number, int, RoundingMode)
206      * @since 1.5.5
207      */
208     public static BigDecimal getDivideValue(Number one,Number two,int scale){
209         return getDivideValue(one, two, scale, HALF_UP);
210     }

212     /**
213      * 获得除法结果<code>one/two</code>,指定舍入方式 <code>roundingMode</code> 以及 小数位数 <code>scale</code>.
214      * 
215      * <h3>示例:</h3>
216      * 
217      * <blockquote>
218      * 
219      * <pre class="code">
220      * NumberUtil.getDivideValue(0, 2, 0,RoundingMode.HALF_UP)   =   0
221      * NumberUtil.getDivideValue(6, 4, 0,RoundingMode.HALF_UP)   =   2
222      * NumberUtil.getDivideValue(10, 3, 2,RoundingMode.HALF_UP)  =   3.33
223      * NumberUtil.getDivideValue(5, 3, 2,RoundingMode.HALF_UP)   =   1.67
224      * </pre>
225      * 
226      * </blockquote>
227      * 
228      * @param one
229      *            除数
230      * @param two
231      *            被除数,自动转成{@link BigDecimal}做除法运算
232      * @param scale
233      *            标度,小数的位数,四舍五入,用于 {@link java.math.BigDecimal#setScale(int, RoundingMode)}<br>
234      *            如果为零或正数,则标度是小数点后的位数。<br>
235      *            如果为负数,则将该数的非标度值乘以 10 的负 scale 次幂 (通常情况用不到负数的情况)
236      * @param roundingMode
237      *            舍入法 {@link RoundingMode}
238      * @return 如果 <code>one</code> 是 null,抛出 {@link NullPointerException}<br>
239      *         如果 <code>two</code> 是 null,抛出 {@link NullPointerException}<br>
240      *         如果 <code>two</code> 是 0,抛出 {@link IllegalArgumentException}<br>
241      *         否则转换成{@link BigDecimal} 返回除法结果one/two,依据舍入法 <code>roundingMode</code>,小数位数 <code>scale</code> 指定
242      * @see <a href="#RoundingMode">JAVA 8种舍入法</a>
243      * @see java.math.BigDecimal#divide(BigDecimal, int, RoundingMode)
244      * @since 1.5.5
245      */
246     public static BigDecimal getDivideValue(Number one,Number two,int scale,RoundingMode roundingMode){
247         Validate.notNull(one, "one can't be null!");
248         Validate.notNull(two, "two can't be null!");
249 
250         BigDecimal divisor = toBigDecimal(two);
251         Validate.isTrue(!divisor.equals(ZERO), "two can't be zero!");
252 
253         // 不能直接one.divide(two),应该指定scale和roundingMode,保证对于无限小数有足够的范围来表示结果.
254         // 避免 exception:Non-terminating decimal expansion; no exact representable decimal result (无法结束的除法表达式；没有精确的除结果)
255         return toBigDecimal(one).divide(divisor, scale, roundingMode);
256     }

262     /**
263      * 将第一个数字 <code>one</code> 和第二个数字 <code>two</code> 相乘,指定精度 <code>scale</code>, 返回{@link BigDecimal}.
264      * 
265      * <h3>示例:</h3>
266      * 
267      * <blockquote>
268      * 
269      * <pre class="code">
270      * NumberUtil.getMultiplyValue(5, 2, 5)                         =   toBigDecimal("10.00000")
271      * NumberUtil.getMultiplyValue(new BigDecimal(6.25), 1.17, 5)   =   toBigDecimal("7.31250")
272      * NumberUtil.getMultiplyValue(9.86, 100, 0)                    =   toBigDecimal("986")
273      * </pre>
274      * 
275      * </blockquote>
276      * 
277      * @param one
278      *            乘数
279      * @param two
280      *            被乘数
281      * @param scale
282      *            标度,小数的位数,四舍五入,用于 {@link java.math.BigDecimal#setScale(int, RoundingMode)}<br>
283      *            如果为零或正数,则标度是小数点后的位数。<br>
284      *            如果为负数,则将该数的非标度值乘以 10 的负 scale 次幂 (通常情况用不到负数的情况)
285      * @return 如果 <code>one</code> 是 null,抛出 {@link NullPointerException}<br>
286      *         如果 <code>two</code> 是 null,抛出 {@link NullPointerException}<br>
287      *         否则 convert to {@link BigDecimal} and multiply each other
288      * @see #setScale(Number, int)
289      * @since 1.5.5
290      */
291     public static BigDecimal getMultiplyValue(Number one,Number two,int scale){
292         Validate.notNull(one, "one can't be null!");
293         Validate.notNull(two, "two can't be null!");
294         //默认返回的精度: (this.scale() + multiplicand.scale()).
295         BigDecimal multiplyValue = toBigDecimal(one).multiply(toBigDecimal(two));
296         return setScale(multiplyValue, scale);
297     }

302     /**
303      * 所有数加起来.
304      * 
305      * <h3>示例:</h3>
306      * 
307      * <blockquote>
308      * 
309      * <pre class="code">
310      * NumberUtil.getAddValue(2, 4, 5)              =   11
311      * NumberUtil.getAddValue(new BigDecimal(6), 5) =   11
312      * </pre>
313      * 
314      * </blockquote>
315      * 
316      * @param numbers
317      *            the numbers
318      * @return 如果 <code>numbers</code> 是null,抛出 {@link NullPointerException}<br>
319      *         如果 <code>numbers</code> 是empty,抛出 {@link IllegalArgumentException}<br>
320      *         如果有元素是null,抛出 {@link IllegalArgumentException}<br>
321      *         否则将每个元素转换成{@link BigDecimal},并进行累加操作
322      * @since 1.5.5
323      */
324     public static BigDecimal getAddValue(Number...numbers){
325         Validate.notEmpty(numbers, "numbers can't be null/empty!");
326         Validate.noNullElements(numbers, "numbers can't has null element!");
327 
328         BigDecimal sum = ZERO;
329         for (Number number : numbers){
330             sum = sum.add(toBigDecimal(number));
331         }
332         return sum;
333     }

335     /**
336      * 所有数相减.
337      * 
338      * <h3>示例:</h3>
339      * 
340      * <blockquote>
341      * 
342      * <pre class="code">
343      * NumberUtil.getSubtractValue(0, 2, 3)                 =   -5
344      * NumberUtil.getSubtractValue(0,null)                  =   0
345      * NumberUtil.getSubtractValue(0,  new Integer[5])      =   0
346      * NumberUtil.getSubtractValue(2, 1.1)                  =   0.9
347      * NumberUtil.getSubtractValue(1000, 99.5, 99.0)        =   801.5
348      * NumberUtil.getSubtractValue(1000, 50, null)          =   950
349      * NumberUtil.getSubtractValue(-1000, -50, 100)         =   -1050
350      * 
351      * NumberUtil.getSubtractValue(null, 5) // NullPointerException
352      * </pre>
353      * 
354      * </blockquote>
355      *
356      * @param beSubtractedValue
357      *            被减数,如 100-10-5, 其中的100 就是被减数
358      * @param subtractions
359      *            减数,如 100-10-5, 其中的10 和5 就是减数
360      * @return 如果 <code>beSubtractedValue</code> 是null,抛出 {@link NullPointerException}<br>
361      *         如果 <code>subtractions</code> 是null或者是empty,直接将<code>beSubtractedValue</code> 转成BigDecimal 并返回<br>
362      *         循环 <code>subtractions</code> ,使用 <code>beSubtractedValue</code>减去元素的值,如果循环的原始null,跳过这次减法<br>
363      * @since 1.10.6
364      */
365     public static BigDecimal getSubtractValue(Number beSubtractedValue,Number...subtractions){
366         Validate.notNull(beSubtractedValue, "beSubtractedValue can't be null/empty!");
367 
368         BigDecimal result = toBigDecimal(beSubtractedValue);
369         if (isNullOrEmpty(subtractions)){
370             return result;
371         }
372 
373         //---------------------------------------------------------------
374         for (Number subtraction : subtractions){
375             if (null == subtraction){//跳过 null 元素
376                 continue;
377             }
378             result = result.subtract(toBigDecimal(subtraction));
379         }
380         return result;
381     }

437     /**
438      * 计算进度(当前量 <code>current</code>/总量 <code>total</code>,然后转成指定的字符串格式 <code>toStringPattern</code>).
439      * 
440      * <p>
441      * 常用于友好的显示 <b>下载进度</b>,<code>执行进度</code>等等场景
442      * </p>
443      * 
444      * <h3>示例:</h3>
445      * 
446      * <blockquote>
447      * 
448      * <pre class="code">
449      * 
450      * NumberUtil.getProgress(5, 5, NumberPattern.PERCENT_WITH_NOPOINT) = "100%"
451      * NumberUtil.getProgress(2, 3, NumberPattern.PERCENT_WITH_1POINT)  = "66.7%"
452      * 
453      * </pre>
454      * 
455      * </blockquote>
456      * 
457      * @param current
458      *            当前量
459      * @param total
460      *            总量
461      * @param toStringPattern
462      *            转成字符串格式 {@link NumberPattern}
463      * @return 如果 <code>current</code> 是null,抛出 {@link NullPointerException}<br>
464      *         如果 <code>total</code> 是null,抛出 {@link NullPointerException}<br>
465      *         如果 {@code current <= 0},抛出 {@link IllegalArgumentException}<br>
466      *         如果 {@code total <= 0},抛出 {@link IllegalArgumentException}<br>
467      *         如果 {@code current > total},抛出 {@link IllegalArgumentException}<br>
468      *         如果 <code>toStringPattern</code> 是null,抛出 {@link NullPointerException}<br>
469      *         如果 <code>toStringPattern</code> 是blank,抛出 {@link IllegalArgumentException}<br>
470      * @see NumberPattern
471      * @see #getDivideValue(Number, Number, int)
472      * @see #toString(Number, String)
473      * @since 1.0.7
474      */
475     public static String getProgress(Number current,Number total,String toStringPattern){
476         Validate.notNull(current, "current can't be null/empty!");
477         Validate.notNull(total, "total can't be null/empty!");
478 
479         Validate.isTrue(current.intValue() > 0, "current can not <=0");
480         Validate.isTrue(total.intValue() > 0, "total can not <=0");
481         Validate.isTrue(current.doubleValue() <= total.doubleValue(), "current can not > total");
482 
483         // XXX  scale = 8不是最优方案
484         int scale = 8;
485         BigDecimal divideValue = getDivideValue(current, total, scale);
486         return toString(divideValue, toStringPattern);
487     }

492      * 使用四舍五入 {@link RoundingMode#HALF_UP} 设置小数点位数.
493      * 
494      * <h3>示例:</h3>
495      * 
496      * <blockquote>
497      * 
498      * <pre class="code">
499      * NumberUtil.setScale(5, 5)        = toBigDecimal("5.00000")
500      * NumberUtil.setScale(5.2, 3)      = toBigDecimal("5.200")
501      * NumberUtil.setScale(5.26, 1)     = toBigDecimal("5.3")
502      * NumberUtil.setScale(-5.26, 1)    = toBigDecimal("-5.3")
503      * 
504      * NumberUtil.setScale(-0, 1)       = toBigDecimal("0.0")
505      * 
506      * NumberUtil.setScale(0, 1)        = toBigDecimal("0.0")
507      * NumberUtil.setScale(0, 2)        = toBigDecimal("0.00")
508      * </pre>
509      * 
510      * </blockquote>
511      * 
512      * <h3>说明:</h3>
513      * <blockquote>
514      * <ol>
515      * <li>被舍入部分{@code >=}0.5向上,否则向下<br>
516      * </li>
517      * <li>{@link RoundingMode#HALF_UP} -2.5 会变成-3,如果是 Math.round(-2.5) 会是-2</li>
518      * </ol>
519      * </blockquote>
520      * 
521      * @param value
522      *            number
523      * @param scale
524      *            标度,小数的位数,四舍五入,用于 {@link java.math.BigDecimal#setScale(int, RoundingMode)}<br>
525      *            如果为零或正数,则标度是小数点后的位数。<br>
526      *            如果为负数,则将该数的非标度值乘以 10 的负 scale 次幂 (通常情况用不到负数的情况)
527      * @return 如果 <code>value</code> 是null,抛出 {@link NullPointerException}<br>
528      * @see <a href="#RoundingMode">JAVA 8种舍入法</a>
529      * @see java.math.RoundingMode#HALF_UP
530      * @see java.math.BigDecimal#ROUND_HALF_UP
531      * @since 1.8.6
532      */
533     public static BigDecimal setScale(Number value,int scale){
534         return setScale(value, scale, HALF_UP);
535     }

537     /**
538      * 使用<code>roundingMode</code> 来设置小数点位数.
539      * 
540      * <h3>示例:</h3>
541      * 
542      * <blockquote>
543      * 
544      * <pre class="code">
545      * NumberUtil.setScale(5, 5,HALF_UP)        = toBigDecimal("5.00000")
546      * NumberUtil.setScale(5.2, 3,HALF_UP)      = toBigDecimal("5.200")
547      * NumberUtil.setScale(5.26, 1,HALF_UP)     = toBigDecimal("5.3")
548      * NumberUtil.setScale(-5.26, 1,HALF_UP)    = toBigDecimal("-5.3")
549      * 
550      * NumberUtil.setScale(-0, 1,HALF_UP)       = toBigDecimal("0.0")
551      * 
552      * NumberUtil.setScale(0, 1,HALF_UP)        = toBigDecimal("0.0")
553      * NumberUtil.setScale(0, 2,HALF_UP)        = toBigDecimal("0.00")
554      * 
555      * NumberUtil.setScale(5, 5,null)           = toBigDecimal("5.00000")
556      * NumberUtil.setScale(5.2, 3,null)         = toBigDecimal("5.200")
557      * NumberUtil.setScale(5.26, 1,null)        = toBigDecimal("5.3")
558      * NumberUtil.setScale(-5.26, 1,null)       = toBigDecimal("-5.3")
559      * 
560      * NumberUtil.setScale(-0, 1,null)          = toBigDecimal("0.0")
561      * 
562      * NumberUtil.setScale(0, 1,null)           = toBigDecimal("0.0")
563      * NumberUtil.setScale(0, 2,null)           = toBigDecimal("0.00")
564      * </pre>
565      * 
566      * </blockquote>
567      * 
568      * @param value
569      *            number
570      * @param scale
571      *            标度,小数的位数,四舍五入,用于 {@link java.math.BigDecimal#setScale(int, RoundingMode)}<br>
572      *            如果为零或正数,则标度是小数点后的位数。<br>
573      *            如果为负数,则将该数的非标度值乘以 10 的负 scale 次幂 (通常情况用不到负数的情况)
574      * @param roundingMode
575      *            舍入模式{@link RoundingMode},如果 为null,使用常用的 {@link RoundingMode#HALF_UP}
576      * @return 如果 <code>value</code> 是null,抛出 {@link NullPointerException}<br>
577      *         如果 <code>roundingMode</code>是null,使用常用的 {@link RoundingMode#HALF_UP} <br>
578      * @see <a href="#RoundingMode">JAVA 8种舍入法</a>
579      * @since 1.8.6
580      */
581     public static BigDecimal setScale(Number value,int scale,RoundingMode roundingMode){
582         Validate.notNull(value, "value can't be null!");
583         return toBigDecimal(value).setScale(scale, defaultIfNull(roundingMode, HALF_UP));
584     }
